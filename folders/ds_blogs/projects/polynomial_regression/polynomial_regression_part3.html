from sklearn.metrics import mean_squared_error
from sklearn.model_selection import train_test_split

def plot_learning_curves(model, X, y):
    X_train, X_val, y_train, y_val = train_test_split(X, y, test_size=0.2, random_state=10)
    train_errors, val_errors = [], []
    for m in range(1, len(X_train)):
        model.fit(X_train[:m], y_train[:m])
        y_train_predict = model.predict(X_train[:m])
        y_val_predict = model.predict(X_val)
        train_errors.append(mean_squared_error(y_train[:m], y_train_predict))
        val_errors.append(mean_squared_error(y_val, y_val_predict))

    plt.plot(np.sqrt(train_errors), "r-+", linewidth=2, label="train")
    plt.plot(np.sqrt(val_errors), "b-", linewidth=3, label="val")
    plt.legend(loc="upper right", fontsize=14)   
    plt.xlabel("Training set size", fontsize=14) 
    plt.ylabel("RMSE", fontsize=14)

Learning Curve for Simple Linear Regression.

    Training curve
  • The error (RMSE) is 0 when the training is a few instances, which means the model can perfectly fit the data.

  • As new training instances were added, the model is no longer able ot fit the training data, especially as the data is not linear and the data is noise, especially at the two extreme of the data distribution.

  • The training data error goes up and plateanus at around 5.0 at 15-20 instances, and no additional data alters the average RMSE. The model can no longer learn from the training.

    Validation curve
  • When the model is trained with a few training instances, the model cannot generalize well so the RMSE remains high.

  • As the training set size increases, the model learns from the data, and the RMSE goes down.

  • Since the model is based on a simple linear regression (straight line), it cannot do a good job to model the actual data, and the error plateaus at a high RMSE above 7.5

  • The training and leanring curves remain close, but the distance between them suggest the model complexity could be improved.

from sklearn.pipeline import Pipeline
plt.figure(figsize=(15,8))
polynomial_regression = Pipeline([
        ("poly_features", PolynomialFeatures(degree=2, include_bias=False)),
        ("lin_reg", LinearRegression()),
    ])

plot_learning_curves(polynomial_regression, lin_x, lin_y)
plt.axis([0, 100, 0, 20]) 
plt.show()

Learning Curve for Polynomial Linear Regression..

    Training curve
  • The error (RMSE) is 0 when the training is a few instances, which means the model can perfectly fit the data.

  • As new training instances were added, the model is no longer able ot fit the training data, especially as the data is not linear and the data is noise, especially at the two extreme of the data distribution.

  • The training data error goes up and plateanus below 5.0 at 15-20 instances, and no additional data alters the average RMSE. The model can no longer learn from the training, but the model RMSE is lower than the simple linear regression, suggesting this model has improved performance.

    Validation curve
  • When the model is trained with a few training instances, the model cannot generalize well so the RMSE remains high. Because this model is more complex than the simple linear regression based on a straight line, the RMSE is higher as the model is trained.

  • As the training set size increases, the model learns from the data, and the RMSE goes down.

  • Since the model is more complex using polynomial regression, the RMSE plateaus at a lower value (~7.0) than the simple linear regression.

  • The training and leanring curves remain close, but the distance between them suggest the model could be improved. The features needs to be either improved, and/or the data needs to be cleaned up.
