Classes and Object-Oriented Programming (OOP)

  • Object oriented programming is one of the many ways to organize source code intended to perform the same set of functions, characteristics, and behaviors.

  • Instead of defining data structures and methods that operate on them separately, OOP explictly groups the code together into an object.

  • This grouping makes the code reusable, scalable, and efficient

  • The principle of explicitly grouping code together is referred to as encapsulation

  • Another way of thinking about this collection code is as an object

  • An object is an entity that encapsulates variables, and a set of functions or methods that operate on data.

  • An example of on object in python programming is a Pandas object.

  • A Pandas object is a structured array in which rows and columns are identified with labels rather than just simple interger indices.

  • Pandas also contains a set of methods (functions) that manipulate the data it contains

  • An object is reusable, and each time the object is used, an instance of that object is created.

  • Each instance of an object is called a class

  • Objects can interact with each other, for example, numpy objects can interact with pandas objects.

  • There are four major principles in Object-Oriented Programming, and they define how objects exist, and how they operate with one another. The include the following:
    1. Encapsulation: object's code are privately held inside its defined boundary. Other objects can interact with this object using public functions or methods, but cannot change the object's source code. This principle provides program security and avoid unintended data corruption.

    2. Abstraction: Objects only reveal necessary information to other objects, so as to avoid sharing any uncesssary implementation code. Why?, because objects can become extremely large, and when objects interact, it is efficient to restrict only relevant information, and not everything the object has under the hood. Example, in pandas, you can calculate the mean of data in a column, it does this by using dataframe.column_name.mean(). The object pandas, interacts with the numpy object, and evokes a specific function, called the mean() to calculate the avereage for a given column in the pandas object. Abstraction lets numpy only share the mean function in this call to pandas, without providing every other function under its hood. Everything else is irrelevant for this specific task.

    3. Inheritance: Many objects are very similar, but not entirely the same. To avoid uncessary duplication of code, new objects can be created from existing objects, this is done by creating a child class from the parent class. This creates a hiearchy, where the child inherits the properties and structure of the parent, but can also have its own unique variables, functions, and methods.

    4. Polymorphism means many shapes. In object oriented programming, it is useful to call the same method to different kinds of objects, and the behavior of that method will differ based on the object. An example in Python would be the addition operator. When used with interger data types, the output will be the sum of the integers. When used with strings, the output is a concatenation of those two strings.

  • Object-Oriented Programming is an important model that has been popular since Alan Kay proposed it back in the 1970s, and is a fundamental basis to languages such as Java, Python, C++, and C#.

class Car:
    
    # Class-Level Attributes/Variables (Data)
    break_speed = -5 # speed that is slowed down by hitting breaks
    
    #Initialise variables for first iteration
    car_nums  = 0
    num_passengers = 0    
    
    # Constructor
    def __init__(self, current_year, car_year, make, model, current_speed, passengers):
    
    # Instance-Level Attributes/Variables (Data)
        self.current_year = current_year # current year
        self.car_year = car_year # year of car model
        self.make = make # car make
        self.model = model # car model
        self.current_speed = current_speed # current speed of vehicle
        Car.car_nums = Car.car_nums + 1
        Car.num_passengers = Car.num_passengers + passengers
            
    # Class Methods
    def age_of_car(self):
        return self.current_year - self.car_year

    def speed_after_brake(self):
        return self.current_speed + self.break_speed

    def make_model(self):
        print('The make is {} and the model is {}'.format(self.make, self.model))

    @classmethod
    def num_of_cars(cls):
        return cls.car_nums

    @classmethod
    def ave_passengers(cls):
        return (round(cls.num_passengers/cls.car_nums, 1))

Anatomy of Class Explained

  • class Car:
    • Defines a class using the class statement

    • The class identifier is typically in CapCase.

    • The body of the class is everything after the colon, and the code is indented.

  • Constructor
    • Is a special method that initializes or activates attributes or properties of the class.

    • The keyword __init__ uses the class methods to create the object.

    • A class can only have one __init__

    • self represents the object that will inherit those the properties initialized.
    • __init__ is called when an object of the class is constructed (instantiated)


  • Class-Level Attributes/Variables
    • Data that is shared among all instances of the class

    • 'Global variable' within the class

    • Does not use self to define class attribute

    • To access attribute, use ClassName.ATTR_NAME, which would be Car.surface_mapg

    • Commonly used for global constants related to the class, if this were a cicle object, we would create a class attribute for pi

  • Instance-Level Attributes/Variables
    • Data that is specifically bound to an instance of the class.

    • Variables are ony accessible after the object is created

    • self binds to an instance.

    • Variables are considered local variables
    • Example: car1 = Car(2017, 'Honda', 'Civic')

    • self will bind the instance attribues in the Car() parameters to the class instance car1

  • Class Methods
    • Functions used in this class that are shared by all instances of this class.

    • There are three types of methods: instance methods, class methods, and static methods

    • Class methods cannot use instance-level data, so self.year cannot be used in this example for its methods. These methods take cls as the first argument, and to access the class variable within the method, @classmethod decorator is passed to the class to the method.

    • Instance methods take self as the first argumnet, and are also called object or regular method

    • Static methods do not take anything as the first argument and are limited in their use

    1. Instance methods
    2. Class methods
    3. Static methods

  • Instance methods take self as the first argument, and are also called object or regular method

  • Dunder methods begin and end with a double underscore

  • __init__, __str__(), and __call__ are examples of the many dunder methods that exist in python.

  • Dunder methods (the use of double underscores) were created so that method names such as call or iter were left available for python programmers, while not having to introduce some new syntax. In other words, by using __call__ instead of call left call available for programmers to use in their code, and this is true for all other dunder methods names.

Inheritance

  • Process by which one class takes on the attributes and methods of another class.

  • The newly formed class is a child of the original parent class.

  • Child classes can override and extend the attributes and methods of the parent class.

  • Child classes can also specify unique attributes and methods.

Polymorphism

  • Polymorphism means many shapes.

  • In object oriented programming, it is useful to call the same method to different kinds of objects, and the behavior of that method will differ based on the object.

  • An example in Python would be the addition operator. When used with interger data types, the output will be the sum of the integers. When used with strings, the output is a concatenation of those two strings.

Data Encapsulation: restricting access to methods and variables

  • This principle prevents the data from being modified by mistake.

  • The use of two understores'__' before the attribute makes the attribute private, and prevents the attribute from being accessible outside the class.

  • If no underscores are used before the attribute, then it is a public attribute.

  • Private attributes can't be accessed outside the class, see example below.

Using @property decorator to change behavior of methods

class Vehicle:
    def __init__(self, driver, make, model, color):
        self.driver = driver # driver name
        self.make = make # car make
        self.model = model # car model
        self.color = color # car color
        
    @property
    def email(self):
        return self.driver +"@datasleuth.com" 
    
    @email.setter
    def email(self, address):
        first = address[:address.find('@')]
        self.driver = first
