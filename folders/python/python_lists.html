Background Info

Python Lists: are one of 6 python built-in sequences. Lists in particular, are ordered collection of objects. Unlike other programming languages, python lists can contain elements of various data types.

Why is this important?
Lists are a ubiquitous and versatile datatype that have the following operations available:

  • indexing
  • slicing
  • adding
  • multiplying
  • checking for membership
  • additional functions

List Intro

Python lists are created by enclosing a comma-separated list of elements in a square bracket. The number of elements in a list can be determined using the built-in len() function.

List Indices

  • Python lists are ordered and each element has an associated index.
  • The index in python starts at 0.
  • List indices can be positive and negative. Negative changes the start point position at the end of the list.
  • The position at the end of the list is -1.

List Slicing

Definition: The process of extracting or assigning a subset of a list.

Slice will extract all elements between index elem1 up to, but not including, index at elem2.

Slice will extract all elements starting at index elem1 up to the end of the list.

Slice will extract all elements from the start of the list, up to, but not including, index at elem2.

Slice will extract all elements from the start of the list, to the end of the list.

Modifying Lists

Definition: The process of changing a single, or multiple elements in a list. General form: 
x[index] = {new value}
    Common methods and statements
  • .append() will add a single element to a list.
  • .extend() will add one list to another.
  • .insert() will add new list elements between two existing elements in a list.
  • .remove() will remove the first instance of an element in a list. Needs exception-handling in cases that an element is missing, otherwise, an error will be raised.
  • .reverse() will reverse a list in place.
  • del list[index] will remove an element at a given index.

Sorting Lists

Definition: The process of arranging list according to characteristics of its elements.

    Common methods and statements
  • .sort() will sort a list in place.
  • .sort(reverse=True) sort has an optional parameter, "reverse" that will arrange the elements by descending value.
  • .sorted() will sort a list without changing the original list.
  • .sort() will work on many datatypes, and even lists of lists, but the list must contain items of comparable types, otherwise an error will be raised.

List Membership

Definition: A test used to determine if a value is in a list, and returns True or False. General form: 
value in list

List Concatenation

Definition: the process of combining two lists, often done using the "+" list concatenation operator. General form: 
list = list1 + list2  => list = [list1, list2]

List initialization with * operator

Definition: the process of producing and initializing a list of a given size using the list multiplication operator "*". General form: 
list = [X] * 3 => list = [X, X, X]

Find smallest and largest values

Definition: the process of finding the largest and smallest elements in a list. General form: 
min(list)
returns the smallest value General form: 
max(list)
returns the largest value An error will be raised if a list does not contain comparable data types.

List Search with Index

Definition: the process of finding the index of a given element. This method searches through the list for an equivalent element, and return the position of that element in the list. General form: 
.index()
General form: 
list.index(value)
An error will be raised if a list does not contain the value in question.

Count

Definition: method that returns the number of times an element is present in a given list. General form: 
list.count(element)
An error will be raised if a list does not contain the value/element in question.

Nested Lists

Definition: a list with lists as elements.
Two dimensional matrices are prime examples of nested lists. General form: 
List[list_a, list_b]
When making copies of nested lists, import copy and then use the copy.deepcopy() function, as other "shallow" copy methods do not create nested lists that are independent of their original list.
