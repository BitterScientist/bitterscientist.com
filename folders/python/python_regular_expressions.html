Using Raw Strings in Regular Expression


  • Regular expressions uses functions that understand certain character sequences in strings have special meanings in the context of regular expressions.
  • Normal strings also possess special sequences, such as new line: "\n", tab: "\t... etc
  • When attempting to use regex to match or compile specific special characters, you must use raw strings: r"string" and not just "string"
  • Using using normal strings, python will interpret the special characters before regex does, and this will have vastly different responses. See below
  • Bottom line: Use raw strings!
