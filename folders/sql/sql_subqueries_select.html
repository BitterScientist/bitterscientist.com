SQL Basics: Using a Subquery in the SELECT Clause.

First Posted: 2-18-2020
Last Updated: 2-18-2020

Background Info

Subqueries: Are SQL statements that are embedded in another query. In other terms, a subquery is a query that is nested in any part of an sql query, such as in the:

  • SELECT Clause
  • FROM Clause
  • WHERE Clause
  • GROUP BY Clause
Why are subqueries in SELECT clause needed?
Using subqueries in a SELECT clause helps bring in summary values into a detailed dataset.
Note: SELECT subqueries need to return a single value, and when filters are used, they need to be included in both the main and the subquery.
  • Returns a single value, and includes aggregate values to compare to individual values.
  • Mathematical calculations
Why is this important?
Nested queries in the SELECT clause enables you to perform complex queries that would be difficult to conduct otherwise, such as:
  • The capacity to return a single value is an important feature, as in normal SQL queries, you cannot include an aggregate value in an ungrouped sequel query. The subquery in the FROM clause is a work around.
  • Returning an aggregated value is useful for performing calculations on data within your database.

Objective: The goal of this post is to introduce subqueries, and provide examples on how they can be used in the SELECT clause.
