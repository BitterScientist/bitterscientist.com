SQL Basics: Using a Subquery in the FROM Clause.

First Posted: 2-4-2020
Last Updated: 2-4-2020

Background Info

Subqueries: Are SQL statements that are embedded in another query. In other terms, a subquery is a query that is nested in any part of an sql query, such as in the:

  • SELECT Clause
  • FROM Clause
  • WHERE Clause
  • GROUP BY Clause
Why are subqueries in FROM clause needed?
Some queries will require intermediate transformation of data, and subqueries in the FROM clause allows for powerful restructuring and transformation.
This is often the case when you need to:
  • Prefilter data
  • Calculate aggregates of aggregated data
Why is this important?
Nested queries in the FROM clause enables you to perform complex queries that would be difficult to conduct otherwise, such as:
  • Compares groups to summarized values
  • Reshapes data for downstream application
  • Combine data that cannot be joined using standard methods

Objective: The goal of this post is to introduce subqueries, and provide examples on how they can be used in the FROM clause.
