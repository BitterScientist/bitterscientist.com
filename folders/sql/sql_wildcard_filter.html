SQL Basics: Filter data using Wildcard cards.

First Posted: 1-23-2020
Last Updated: 1-23-2020

Background Info

Wildcards: A wildcard is a search pattern (string) used to filter data from a table(s) using characters that have special meanings within SQL WHERE clauses. Unlike other filtering techniques that use known values, the use wildcards allows for sophistical filtering using only a search pattern.
The search pattern is either a wildcard character, a literal text, or a combination of those two, but not a non-text datatype.

Objective: The goal of this post is to highlight some of the advance filtering strategies that can be used when a WHERE clause is combined with a wildcard.

Data: The data for this post was obtained from the state of Connecticut Health And Human Services division, Office of the Chief Medical Examiner. A listing of each accidental death associated with drug overdose in Connecticut from 2012 to 2018. The following description taken from: https://data.ct.gov/Health-and-Human-Services/Accidental-Drug-Related-Deaths-2012-2018/rybz-nyjw Data are derived from an investigation by the Office of the Chief Medical Examiner which includes the toxicity report, death certificate, as well as a scene investigation. https://data.ct.gov/Health-and-Human-Services/Accidental-Drug-Related-Deaths-2012-2018/rybz-nyjw Data retrieved 1-21-20

Method: This post used the python libraries pandas, and SQLAlchemy were used in conjunction with jupyter notebook to run the MySQL SELECT commands. This was done for ease of displaying the results. All SQL commands are in red.
