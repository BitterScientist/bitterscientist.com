Structured Query Language (SQL)

Background Info

Structured Query Language: is a domain specific language used to design, and manage data that is held in a relational database. Unlike Java, C, or PHP, the language of SQL consists of a small library of words, and this is to make it simple, and efficient to read, and write. It is the most popular database language today, and was initially developed by IBM, after Edgar Codd 1970s seminal publication on "A relational model of data for large shared databanks".

Why is this important?
Relational databases are a ubiquitous element in our lives, and powers much of the technology we use on a daily basis. To this end, having a basic understanding of the language used to design, access, and manage a relational database is essential for analyst, data scientist, web and add developers, software engineers, and anyone with an interest in technology.

The advantages of SQL:

  • Not a proprietary language, therefore it is used by all major Relational Database Management Systems (RDBMS). In other words, you just need to learn one SQL language to use all major database systems.
  • Learning SQL is simple, the language consists of a small set of words, and all the statement used are descriptive and in English.
  • SQL is a powerful that allows users to conduct sophisticated database operations. That is the goal of these posts, to share with you these operations.

Comments on RDBMS language extensions: If you're working what's the difference between PL-SQL, T-SQL and so forth, its the individual implementations these RDMS. SQL though is governed by ANSI SQL, and all RDMS support ANSI SQL. The RDMS language extensions are intended to provide additional functionality, and you would need to dig into their documentations to understand their operations.
