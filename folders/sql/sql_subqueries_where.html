Background Info

Subqueries: Are SQL statements that are embedded in another query. In other terms, a subquery is a query that is nested in any part of an sql query, such as:

  • SELECT
  • FROM
  • WHERE
  • GROUP BY

Why are subqueries needed?
Some queries will require intermediate transformation of data, and subqueries provides this functionality by conducting a nested query, and returning that data for the main query to use.
Example data that can be returned in a subquery:
  • Scalar values
  • List of values to be used for filtering or joining information

Why is this important?
Nested queries enable you to perform complex queries that:
  • Compares groups to summarized values
  • Reshapes data for downstream application
  • Combines data that cannot be joined using standard methods

Objective: The goal of this post is to introduce subqueries, and provide examples on how they can be used.
